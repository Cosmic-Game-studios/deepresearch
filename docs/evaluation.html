Critical findings

C1: Zero real-world validation

No human or agent has ever used DeepResearch on a real project. Every benchmark runs on synthetic landscapes. The tool has never optimized a real training script, a real API, a real game, or a real prompt. This is the single biggest credibility gap.

IMPACT: Fatal for any SOTA claim

C2: Benchmarks model ideal reasoning, not real LLM behavior

benchmark_reasoning.py models "understanding" as a smooth function reaching 85% after ~10 attempts. Real LLMs hallucinate, misunderstand code, and sometimes get worse with more context. The +14% is an upper bound, not a realistic estimate. The +189% L3 advantage compares "can add features" vs "can only tune" — logically obvious, doesn't prove the engine works.

IMPACT: Claims may not hold in practice

C3: Level 2-3 engine is structurally untested

2,845 lines across 6 modules, 18 classes, 110 functions. Unit tests cover state machines but never test the mutation→test→score→revert cycle on real files. MutationManager.execute() — the core of Level 2 — has never run with real file I/O, real test commands, or real metrics.

IMPACT: Level 2-3 is unproven code

Major findings

M1: SKILL.md is 1,785 lines — too long for most models

~7,000 tokens to read. Sonnet, GPT-4o, and open-source models will struggle to follow all instructions. Phase 1 alone is 445 lines. No "lite" version exists for smaller models.

M2: strategy.py and engine/ are disconnected systems

strategy.py has zero imports from engine/. engine/ has zero from strategy.py. pipeline.py bridges them but is barely referenced (1 mention in SKILL.md). Four separate state files with no unified access.

M3: DomainResearcher overlaps with KnowledgeAcquisition

Both handle "research the domain." Built in different sessions, never unified. An agent doesn't know which to use.

M4: Section imbalance — mechanics outweigh thinking 5:1

Phase 1 (Core Loop): 445 lines (25%). Reasoning Layer: 91 lines (5%). The claimed differentiator gets the least attention.

Strengths

S1: Most complete architecture in the autoresearch ecosystem

7 layers. No competitor has: Reasoning Layer + Knowledge Acquisition + Curriculum + Generative Mutations + Population Search + Persistent Memory + Autonomous Pipeline. All in one system.

S2: The Reasoning Layer concept is genuinely novel

R1→R2→R3 (Deep Read → Causal Hypothesis → Reflection) with concrete good/bad examples. No other tool frames the LLM as a thinking researcher rather than a code editor.

S3: Knowledge Acquisition system is unique

753 lines: SearchStrategy, SourceManager, TechniqueLibrary, KnowledgeIntegration. Nothing like it exists in the ecosystem. 33 functions, full test coverage.

S4: 170 tests, all passing

16 test functions with temp directory isolation. Better than most research-stage tools.

Recommendation

Stop building. Start proving.

The architecture is sufficient. Adding more code will not change the score. What changes the score: (1) one real-world result showing the Reasoning Layer beats greedy on actual code, (2) one successful Level 2 run, (3) external users. The difference between 6.2 and 9.0 is not code — it's evidence.
